# E2E テスト記述ルール

## 基本方針

- 操作対象の特定は `data-testid` を優先する
- 画面遷移後は、そのページを識別できる要素の表示まで確認する
- 成功時のログは簡素に保ち、失敗時のエラー出力は残す

## 要素の特定

- 操作対象の要素は `data-testid` で特定する
- `getByText` / `getByRole` は、表示内容そのもののアサーションやエラーメッセージ確認では使用可
- `locator("..")` のような親 DOM 依存のたどり方は避ける

## 本番コードの修正

- `data-testid` の追加は確認不要で行ってよい
- `data-testid` はラッパー要素ではなく、実際に操作・判定したい要素へ付与する

## テストの独立性

- テスト間のデータ汚染を防ぎ、各テストは独立して実行できる状態を保つ
- setup / cleanup は個別 test に散らさず、共通化できる場合は共通化する
- 部分的にデータが残る cleanup より、確実に初期状態へ戻せる方法を優先する

## 規約

- テスト名は日本語で書く
- 共通 fixture がある場合はそれを import する
- コメントは日本語で書く
