# ユーザーフロー設計

## 概要

タスクのライフサイクル（追加→次やる設定→作業開始→完了/中断）の操作フローが未設計。
大まかなフロー案を作成し、確認を得た上でUIデザインに反映する。

## 現状の課題

### 1. 「次やる」以外のタスクから作業を始められない
- 現UIではホーム画面の「次やるタスク」にのみ「作業をはじめる」ボタンがある
- 他のタスクを急遽始めたい場合の導線がない

### 2. 「次やる」タスクの設定方法が不明
- どの画面でどう操作して「次やる」マークを付けるのか未定義
- タスク一覧からタップ？長押し？専用ボタン？

### 3. タスク追加画面がない
- タスクCRUDのUI（追加・編集・削除）が未設計
- 入力項目（タスク名、カテゴリ、予定日、見積もり時間など）も未定義

### 4. 作業開始後の状態遷移が不明
- 「作業をはじめる」を押したらどうなる？
  - タイマー画面に遷移？
  - タイマーの時間はどう決まる？（タスクに設定済み？その場で入力？）

### 5. 完了/中断の操作が不明
- 作業中のタスクを「完了」にする操作は？
  - タイマー終了時のみ？途中でも可能？
- 作業が終わらなかった場合の「中断」操作は？
  - 中断後、タスクはどの状態に戻る？
  - 中断した作業時間はカレンダー（作業記録）に残る？

### 6. タイマーとの連携
- 作業開始 = タイマー開始？
- タイマー終了時の選択肢は？（完了 / 継続 / 中断）
- タイマーなしで作業することは可能？

### 7. PC版タイマーの扱い（別課題だが関連あり）
- 現デザインではPC版にタイマー画面がない
- PCでは別アプリ（インスタ、メモアプリ等）で作業するため、理想はPiP風フローティングタイマー → `docs/issues/002-pc-pip-timer.md`
- **PiPは余裕があれば対応。初期リリースではPC版タイマーを別の簡易な方法で実現する必要あり**
- フロー設計時の注意: PC版の「作業開始→タイマー」の遷移先をどうするか決める必要がある

## 進め方

1. Claude がフロー全体の叩き台を作成
2. ユーザーが質問・訂正
3. フロー確定後、UIデザイン（`designs/todo-app.pen`）に反映
