# ローディング状態のデザイン

## 概要

DB移行時に必要となるローディング状態（スケルトン or ローディングインジケーター）のデザインを決める。

## 背景

- 現状はローカルストレージのため読み込みは数msで体感上問題なし
- DB（Supabase等）移行後は通信待ちが発生するため、ローディング表示が必要になる
- 003-ux-review.md #9 から派生

## 対応タイミング

DB移行時

## 検討事項

- スケルトンスクリーン vs ローディングインジケーター（スピナー等）
- 対象画面: ホーム、タスク一覧、作業記録、PC各画面
- 300ms以上の待機にはフィードバックが必要（UXベストプラクティス）
