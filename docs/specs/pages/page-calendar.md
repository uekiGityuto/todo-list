# カレンダー画面 仕様

## パス

`/calendar`

## 概要

日付ごとの作業記録を表示。完了・中断した作業の履歴確認に使う。

## レイアウト

### モバイル

```
[header]              ← "作業記録" タイトル + 月ナビ（< 2月 2026 >）
[calendarGrid]
  [dayHeaders]        ← 日〜土の曜日ヘッダー
  [gridBody]          ← CalendarCell のグリッド（4〜5週）
[dayDetail]
  [SectionHeader]     ← "X月X日の作業" + "N件の作業"
  [TaskCard] x N      ← その日に完了/中断したタスク
[TabBar]              ← activeTab="calendar"
```

### PC

```
[Sidebar activeItem="calendar"] | [mainArea]
                                |   [header + 月ナビ]
                                |   [calendarGrid]  ← PC版: セル内にタスク名表示
                                |   [dayDetail]
```

## 使用コンポーネント

| コンポーネント       | 用途                                                    |
| -------------------- | ------------------------------------------------------- |
| `CalendarCell`       | カレンダーの各日セル（モバイル: ドット / PC: タスク名） |
| `SectionHeader`      | 日付詳細の見出し                                        |
| `TaskCard`           | その日の完了/中断タスク表示（チェック済み状態）         |
| `Check`              | TaskCard 内（完了: Done、中断: Empty）                  |
| `TabBar` / `Sidebar` | ナビゲーション                                          |

## カレンダーグリッド

- 曜日ヘッダー: 日・月・火・水・木・金・土（各 fill_container 幅）
- グリッド: `border` 色で1pxのセパレータ
- セルサイズ: モバイル h-16 / PC h-20
- 今日の日付: `primary` 色 + bold

### モバイル表示

- セル内にドット表示（最大3個、`primary` 色の丸 6px）

### PC表示

- セル内にタスク名テキスト表示（最大2件、`primary` 色 12px）

## 状態

### 通常状態

- カレンダーグリッド + 選択日の作業詳細

### 空状態（選択日に作業なし）

- "実施したタスクはありません"（14px, `text-muted` 色）

### 月ナビゲーション

- 「<」「>」で前月/翌月に切り替え
- 中央に "X月 YYYY" 表示

## インタラクション

| 操作                | 動作                         |
| ------------------- | ---------------------------- |
| CalendarCell タップ | その日の作業詳細を下部に表示 |
| 月ナビ「<」「>」    | 前月/翌月に切り替え          |
| TabBar / Sidebar    | ページ遷移                   |

## 実装時の考慮事項

### デフォルト選択日

- 初期表示時は **今日の日付** を選択状態にする
- 選択日にタスク（作業記録）がなければ空状態メッセージを表示

### オーバーレイ

- モーダル等の浮き要素には `bg-overlay`（`#00000040`）を適用

## データ要件

- 月ごとの作業記録（日付 → タスク一覧のマッピング）
- 各記録: タスク名、カテゴリ、作業時間、完了/中断ステータス
